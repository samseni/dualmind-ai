## 📄 File-by-File Breakdown

### 1. `.gitignore`

**Location:** `frontend/.gitignore`

**Purpose:** Tells Git which files and folders to ignore (not track in version control)

**What it does:**
- Prevents `node_modules/` from being committed (too large, can be reinstalled)
- Ignores `dist/` folder (build output, generated files)
- Ignores editor files (`.vscode`, `.idea`, `.DS_Store`)
- Ignores log files and temporary files

**Why it's important:**
- Keeps repository clean and small
- Prevents sensitive files from being accidentally committed
- Ensures only source code is tracked, not generated files

**Key contents:**
```
node_modules/      # NPM dependencies (reinstall with npm install)
dist/              # Build output (generated by npm run build)
.env               # Environment variables (if you add any)
.DS_Store          # macOS system files
```

### 2. `index.html`

**Location:** `frontend/index.html`

**Purpose:** The HTML file that loads in the browser (entry point for the app)

**What it does:**
- Provides the basic HTML structure
- Creates a `<div id="root"></div>` where React will render
- Loads the React app via `<script src="/src/main.jsx">`
- Sets the page title and meta tags

**Key sections:**
```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>DualMind AI - Chat with Claude & ChatGPT</title>
</head>
<body>
  <div id="root"></div>                    <!-- React renders here -->
  <script type="module" src="/src/main.jsx"></script>  <!-- Loads React app -->
</body>
```

**Flow:**
1. Browser loads `index.html`
2. Browser sees `<script src="/src/main.jsx">`
3. Vite processes the script and loads React
4. React renders the entire app inside `<div id="root">`

### 4. `vite.config.js`

**Location:** `frontend/vite.config.js`

**Purpose:** Configuration file for Vite (the build tool and dev server)

**What it does:**
- Enables React support via plugin
- Configures the development server
- **Sets up proxy to backend** (routes `/api` requests to backend server)

**Key sections:**

**React Plugin:**
```javascript
plugins: [react()]  // Enables JSX, Fast Refresh, etc.
```

**Server Configuration:**
```javascript
server: {
  port: 5173,        // Frontend runs on http://localhost:5173
  proxy: {
    '/api': {
      target: 'http://localhost:5000',  // Backend URL
      changeOrigin: true,
    }
  }
}
```

**How the proxy works:**
- When frontend makes request to `/api/chat`
- Vite intercepts it and forwards to `http://localhost:5000/api/chat`
- Backend responds, Vite sends response back to frontend
- This avoids CORS (Cross-Origin) issues during development

**Example:**
```javascript
// Frontend code makes this request:
fetch('/api/chat', { ... })

// Vite automatically converts it to:
fetch('http://localhost:5000/api/chat', { ... })
```

### 5. `main.jsx`

**Location:** `frontend/src/main.jsx`

**Purpose:** Entry point for the React application

**What it does:**
- Finds the `<div id="root">` in `index.html`
- Renders the entire React app inside it
- Wraps app in `React.StrictMode` for better error detection

**Code breakdown:**
```javascript
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)
```

**What each line does:**
1. `import React from 'react'` - Import React library
2. `import ReactDOM from 'react-dom/client'` - Import React's browser renderer
3. `import App from './App.jsx'` - Import our main component
4. `document.getElementById('root')` - Find the `<div id="root">` in HTML
5. `ReactDOM.createRoot(...)` - Create a React root at that div
6. `.render(<App />)` - Render the App component
7. `<React.StrictMode>` - Enable extra checks and warnings (development only)

**Flow:**
```
index.html loads → main.jsx runs → App.jsx renders → Full UI appears
```

### 8. `ChatBox.jsx`

**Location:** `frontend/src/components/ChatBox.jsx`

**Purpose:** Main chat interface component (the heart of the app)

**What it does:**
- Manages all chat state (messages, loading, selected model)
- Handles user input and message sending
- Communicates with backend API
- Displays messages and controls

**State variables:**
```javascript
const [messages, setMessages] = useState([]);              // Array of all messages
const [inputValue, setInputValue] = useState('');          // Current text in input box
const [selectedModel, setSelectedModel] = useState('claude');  // Selected AI model
const [isLoading, setIsLoading] = useState(false);        // Is AI responding?
```

**Key functions:**

**1. `handleSendMessage()` - Sends message to backend:**
```javascript
const handleSendMessage = async () => {
  // 1. Add user message to chat
  setMessages(prev => [...prev, userMessage]);

  // 2. Call backend API
  const response = await fetch('/api/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      message: inputValue,
      model: selectedModel,
      conversationHistory: messages,
    }),
  });

  // 3. Get AI response
  const data = await response.json();

  // 4. Add AI message to chat
  setMessages(prev => [...prev, aiMessage]);
}
```

**2. `scrollToBottom()` - Auto-scrolls to latest message:**
```javascript
const scrollToBottom = () => {
  messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' });
};

useEffect(() => {
  scrollToBottom();
}, [messages]);  // Runs every time messages change
```

**3. `handleKeyPress()` - Send on Enter key:**
```javascript
const handleKeyPress = (e) => {
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    handleSendMessage();
  }
};
```

**What it renders:**
1. **Header:** Title and subtitle
2. **Controls:** Model selector + Clear Chat button
3. **Messages area:** All messages or welcome screen
4. **Loading indicator:** Typing animation while AI responds
5. **Input area:** Text box + Send button

**Error handling:**
```javascript
try {
  // Call API
} catch (error) {
  // Show error message in chat
  const errorMessage = {
    role: 'assistant',
    content: `Sorry, I encountered an error: ${error.message}`,
    model: 'System',
  };
  setMessages(prev => [...prev, errorMessage]);
}
```

### 9. `Message.jsx`

**Location:** `frontend/src/components/Message.jsx`

**Purpose:** Displays a single message bubble (user or AI)

**What it does:**
- Receives `role` (user or assistant), `content` (text), and `model` (which AI)
- Renders a styled message bubble
- Shows different icons for user vs AI

**Props:**
```javascript
<Message
  role="user"               // 'user' or 'assistant'
  content="Hello!"          // The actual message text
  model="Claude"            // Which AI model (optional)
/>
```

**Code breakdown:**
```javascript
const Message = ({ role, content, model }) => {
  return (
    <div className={`message ${role}`}>
      <div className="message-header">
        <span className="message-role">
          {role === 'user' ? '👤 You' : `🤖 ${model || 'AI'}`}
        </span>
      </div>
      <div className="message-content">
        {content}
      </div>
    </div>
  );
};
```

**What it renders:**
- **User message:** "👤 You" + message text (purple background)
- **AI message:** "🤖 Claude" or "🤖 ChatGPT" + message text (white background)

**CSS classes:**
- `message` - Base styles for all messages
- `message user` - Styles for user messages (purple, right-aligned)
- `message assistant` - Styles for AI messages (white, left-aligned)

### 10. `ModelSelector.jsx`

**Location:** `frontend/src/components/ModelSelector.jsx`

**Purpose:** Dropdown to select which AI model to use

**What it does:**
- Shows a dropdown with options: Claude or ChatGPT
- Calls parent component when user changes selection
- Displays current selected model

**Props:**
```javascript
<ModelSelector
  selectedModel="claude"                   // Current selection
  onModelChange={(model) => {...}}        // Function to call when changed
/>
```

**Code breakdown:**
```javascript
const ModelSelector = ({ selectedModel, onModelChange }) => {
  return (
    <div className="model-selector">
      <label htmlFor="model-select">AI Model:</label>
      <select
        id="model-select"
        value={selectedModel}
        onChange={(e) => onModelChange(e.target.value)}
      >
        <option value="claude">Claude (Anthropic)</option>
        <option value="chatgpt">ChatGPT (OpenAI)</option>
      </select>
    </div>
  );
};
```

**How it works:**
1. User clicks dropdown
2. User selects "ChatGPT"
3. `onChange` event fires
4. Calls `onModelChange('chatgpt')`
5. Parent component (`ChatBox`) updates `selectedModel` state
6. Next message will be sent to ChatGPT instead of Claude

## 🔄 How Components Work Together

### Component Hierarchy:
```
main.jsx
  └── App.jsx
        └── ChatBox.jsx
              ├── ModelSelector.jsx
              └── Message.jsx (rendered multiple times)
```

### Data Flow:
```
User types message
    ↓
ChatBox.jsx captures input
    ↓
User clicks "Send"
    ↓
ChatBox.jsx calls handleSendMessage()
    ↓
Sends POST request to /api/chat
    ↓
Vite proxy forwards to backend (http://localhost:5000/api/chat)
    ↓
Backend processes request (calls Claude or ChatGPT)
    ↓
Backend sends response
    ↓
ChatBox.jsx receives response
    ↓
Adds AI message to messages array
    ↓
React re-renders
    ↓
New Message.jsx component appears
    ↓
Auto-scrolls to bottom
```

## 🐛 Common Issues & Fixes

### Issue 1: "Cannot find module 'react'"
**Fix:** Run `npm install` in the frontend folder

### Issue 2: API calls failing with 404
**Fix:** Make sure backend is running on port 5000

### Issue 3: Vite proxy not working
**Fix:** Check `vite.config.js` has correct backend URL

### Issue 4: CSS not loading
**Fix:** Make sure `App.css` is imported in `App.jsx`

### Issue 5: White screen / no render
**Fix:** Check browser console for errors, verify `main.jsx` is loaded

## 🎯 Summary

**Each file's job in one sentence:**

1. **`.gitignore`** - Tells Git what to ignore
2. **`index.html`** - HTML entry point that loads React
3. **`package.json`** - Lists dependencies and scripts
4. **`vite.config.js`** - Configures build tool and proxy
5. **`main.jsx`** - Renders React app into HTML
6. **`App.jsx`** - Root React component

8. **`ChatBox.jsx`** - Main chat logic and API calls
9. **`Message.jsx`** - Single message bubble display
10. **`ModelSelector.jsx`** - AI model dropdown selector

**The flow in simple terms:**
1. Browser loads `index.html`
2. HTML loads `main.jsx` (via Vite)
3. `main.jsx` renders `App.jsx`
4. `App.jsx` renders `ChatBox.jsx`
5. `ChatBox.jsx` renders `ModelSelector.jsx` and multiple `Message.jsx` components
6. User types and sends message
7. `ChatBox.jsx` calls backend API
8. Response displayed as new `Message.jsx`
