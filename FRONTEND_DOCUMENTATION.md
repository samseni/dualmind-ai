## 📄 File-by-File Breakdown

### 1. `.gitignore`

**Location:** `frontend/.gitignore`

**Purpose:** Tells Git which files and folders to ignore (not track in version control)

**What it does:**
- Prevents `node_modules/` from being committed (too large, can be reinstalled)
- Ignores `dist/` folder (build output, generated files)
- Ignores editor files (`.vscode`, `.idea`, `.DS_Store`)
- Ignores log files and temporary files

**Why it's important:**
- Keeps repository clean and small
- Prevents sensitive files from being accidentally committed
- Ensures only source code is tracked, not generated files

**Key contents:**
```
node_modules/      # NPM dependencies (reinstall with npm install)
dist/              # Build output (generated by npm run build)
.env               # Environment variables (if you add any)
.DS_Store          # macOS system files
```

### 4. `vite.config.js`

**Location:** `frontend/vite.config.js`

**Purpose:** Configuration file for Vite (the build tool and dev server)

**What it does:**
- Enables React support via plugin
- Configures the development server
- **Sets up proxy to backend** (routes `/api` requests to backend server)

**Key sections:**

**React Plugin:**
```javascript
plugins: [react()]  // Enables JSX, Fast Refresh, etc.
```

**Server Configuration:**
```javascript
server: {
  port: 5173,        // Frontend runs on http://localhost:5173
  proxy: {
    '/api': {
      target: 'http://localhost:5000',  // Backend URL
      changeOrigin: true,
    }
  }
}
```

**How the proxy works:**
- When frontend makes request to `/api/chat`
- Vite intercepts it and forwards to `http://localhost:5000/api/chat`
- Backend responds, Vite sends response back to frontend
- This avoids CORS (Cross-Origin) issues during development

**Example:**
```javascript
// Frontend code makes this request:
fetch('/api/chat', { ... })

// Vite automatically converts it to:
fetch('http://localhost:5000/api/chat', { ... })
```

### 5. `main.jsx`

**Location:** `frontend/src/main.jsx`

**Purpose:** Entry point for the React application

**What it does:**
- Finds the `<div id="root">` in `index.html`
- Renders the entire React app inside it
- Wraps app in `React.StrictMode` for better error detection

**Code breakdown:**
```javascript
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,

```

**What each line does:**
1. `import React from 'react'` - Import React library
2. `import ReactDOM from 'react-dom/client'` - Import React's browser renderer
3. `import App from './App.jsx'` - Import our main component
4. `document.getElementById('root')` - Find the `<div id="root">` in HTML
5. `ReactDOM.createRoot(...)` - Create a React root at that div
6. `.render(<App />)` - Render the App component
7. `<React.StrictMode>` - Enable extra checks and warnings (development only)

**Flow:**
```
index.html loads → main.jsx runs → App.jsx renders → Full UI appears
```

### 6. `App.jsx`

**Location:** `frontend/src/App.jsx`

**Purpose:** Root component of the React application + Theme management

**What it does:**
- Wraps the entire application
- Manages dark/light theme state
- Saves theme preference to localStorage
- Passes theme controls to ChatBox component
- Imports global styles from `App.css`

**Code breakdown:**
```javascript
import React, { useState, useEffect } from 'react';
import ChatBox from './components/ChatBox';
import './App.css';

function App() {
  const [theme, setTheme] = useState(() => {
    const savedTheme = localStorage.getItem('theme');
    return savedTheme || 'dark';
  });

  useEffect(() => {
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('theme', theme);
  }, [theme]);

  const toggleTheme = () => {
    setTheme(prevTheme => prevTheme === 'dark' ? 'light' : 'dark');
  };

  return (
    <div className="App">
      <ChatBox theme={theme} toggleTheme={toggleTheme} />
    </div>
  );
}

export default App;
```

**State management:**
- `theme` - Current theme ('dark' or 'light')
- Loads saved preference from localStorage on mount
- Applies theme via `data-theme` attribute on document root
- Saves changes back to localStorage

**What it renders:**
- A `<div className="App">` wrapper
- The `<ChatBox />` component with theme props

**Why it exists:**
- Centralizes theme management for entire app
- Provides a place for app-wide features (routing, auth, etc.)
- Keeps structure organized and modular

### 8. `ChatBox.jsx`

**Location:** `frontend/src/components/ChatBox.jsx`

**What it does:**
- Manages all chat state (messages, loading, selected model)
- Handles user input and message sending

**State variables:**
```javascript
const [messages, setMessages] = useState([]);              // Array of all messages
```

**Key functions:**

**1. `handleSendMessage()` - Sends message to backend:**
```javascript
const handleSendMessage = async () => {
  // 1. Add user message to chat
  setMessages(prev => [...prev, userMessage]);

  // 2. Call backend API
  const response = await fetch('/api/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      message: inputValue,
      model: selectedModel,
      conversationHistory: messages,
    }),
  });

  // 3. Get AI response
  const data = await response.json();

  // 4. Add AI message to chat
  setMessages(prev => [...prev, aiMessage]);
}
```

**2. `scrollToBottom()` - Auto-scrolls to latest message:**
```javascript
const scrollToBottom = () => {
  messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' });
};

useEffect(() => {
  scrollToBottom();
}, [messages]);  // Runs every time messages change
```

**3. `handleKeyPress()` - Send on Enter key:**
```javascript
const handleKeyPress = (e) => {
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    handleSendMessage();
  }
};
```

**What it renders:**
1. **Header:** Title and subtitle
2. **Controls:** Model selector + Clear Chat button
3. **Messages area:** All messages or welcome screen
4. **Loading indicator:** Typing animation while AI responds
5. **Input area:** Text box + Send button

**Error handling:**

```javascript
const formatDate = (timestamp) => {
  if (!timestamp) return 'Just now';
  const date = new Date(timestamp);
  const now = new Date();
  const diffDays = Math.floor((now - date) / (1000 * 60 * 60 * 24));

  if (diffDays === 0) return 'Today';
  if (diffDays === 1) return 'Yesterday';
  if (diffDays < 7) return `${diffDays} days ago`;
  return date.toLocaleDateString('en-US', { month: 'short', day: 'numeric' });
};
```

**2. `getChatTitle()` - Generates title from first message:**
```javascript
const getChatTitle = (chat) => {
  if (chat.title) return chat.title;
  if (chat.messages && chat.messages.length > 0) {
    const firstMessage = chat.messages[0].content;
    return firstMessage.length > 30
      ? firstMessage.substring(0, 30) + '...'
      : firstMessage;
  }
  return 'New Chat';
};
```

## 🔄 How Components Work Together

### Component Hierarchy:
```
main.jsx
  └── App.jsx (manages theme)
        └── ChatBox.jsx (manages chat state + history)
              ├── ChatHistory.jsx (sidebar)
              │     └── Conversation items
              ├── ModelSelector.jsx (dropdown)
              └── Message.jsx (rendered multiple times)
```

### Data Flow:
```
User types message
    ↓
ChatBox.jsx captures input
    ↓
User clicks "Send"
    ↓
ChatBox.jsx calls handleSendMessage()
    ↓
Sends POST request to /api/chat
    ↓
Vite proxy forwards to backend (http://localhost:5000/api/chat)
    ↓
Backend processes request (calls Claude or ChatGPT)
    ↓
Backend sends response
    ↓
ChatBox.jsx receives response
    ↓
Adds AI message to messages array
    ↓
React re-renders
    ↓
New Message.jsx component appears
    ↓
Auto-scrolls to bottom
```

## 🐛 Common Issues & Fixes

### Issue 1: "Cannot find module 'react'"
**Fix:** Run `npm install` in the frontend folder

### Issue 2: API calls failing with 404
**Fix:** Make sure backend is running on port 5000

### Issue 3: Vite proxy not working
**Fix:** Check `vite.config.js` has correct backend URL

### Issue 4: CSS not loading
**Fix:** Make sure `App.css` is imported in `App.jsx`

## 🎯 Summary

**Each file's job in one sentence:**

1. **`.gitignore`** - Tells Git what to ignore
2. **`index.html`** - HTML entry point that loads React
3. **`package.json`** - Lists dependencies and scripts
4. **`vite.config.js`** - Configures build tool and proxy
5. **`main.jsx`** - Renders React app into HTML
6. **`App.jsx`** - Root React component + theme management
7. **`App.css`** - Global styles, dark/light themes, and sidebar CSS
8. **`ChatBox.jsx`** - Main chat logic, API calls, and chat history management
9. **`Message.jsx`** - Single message bubble with avatar and timestamp
10. **`ModelSelector.jsx`** - AI model dropdown selector with brand logos
11. **`claude.svg` & `chatgpt.svg`** - Theme-adaptive SVG logos for AI models
12. **`ChatHistory.jsx`** - Sidebar with saved conversations list

**The flow in simple terms:**
1. Browser loads `index.html`
2. HTML loads `main.jsx` (via Vite)
3. `main.jsx` renders `App.jsx`
4. `App.jsx` manages theme and renders `ChatBox.jsx`
5. `ChatBox.jsx` renders:
   - `ChatHistory.jsx` (sidebar with saved chats)
   - `ModelSelector.jsx` (AI model dropdown)
   - Multiple `Message.jsx` components (the chat)
6. User types and sends message
7. `ChatBox.jsx` calls backend API
8. Response displayed as new `Message.jsx`
9. Conversation auto-saved to localStorage
10. `ChatHistory.jsx` updates with latest chat
